# Plan 02-02: Real-time Inference Integration

Integrate the trained ML models into the ingestion pipeline to compute anomaly scores and severity levels for incoming vitals.

## Must Haves
- `InferenceEngine` class to load model artifacts and perform scoring.
- Windowing/Buffering logic to aggregate per-sensor messages into full feature vectors.
- Integration with `src/ingestion/worker.py` to add `anomaly_score` and `severity` to records before DB insertion.
- Logic to detect HIGH-severity anomalies and trigger (placeholder) alerts.

## Tasks

<task id="02-02-00" wave="0">
Implement `src/intelligence/inference.py` with `InferenceEngine`.
- Handle model loading (Scaler, Autoencoder, Isolation Forest).
- Implement `score(patient_vitals)` method.
- Implement `process_sensor_data(patient_id, sensor_type, value)` with in-memory buffering.
</task>

<task id="02-02-01" wave="1">
Update `src/ingestion/worker.py` to use `InferenceEngine`.
- Initialize `InferenceEngine` in the worker.
- For each message, call `inference_engine.process_sensor_data()`.
- If a complete feature vector is formed, update the record with the score and severity.
</task>

<task id="02-02-02" wave="2">
Implement `tests/integration/test_inference_pipeline.py`.
- Mock Kafka messages and verify that the pipeline correctly computes scores and severity.
- Verify that records with scores are correctly formatted for DB insertion.
</task>

### Automated Tests
