# Plan 03-01: API Development

Implement the Flask REST API to serve patient vitals, anomaly history, and patient metadata from the TimescaleDB.

## Must Haves
- Flask application structure in `src/api/`.
- Database utility functions to query `vitals` and `anomalies`.
- Endpoints:
  - `/api/patients`: List unique patient IDs.
  - `/api/vitals/<patient_id>`: Get recent vitals (limit 100) for a specific patient.
  - `/api/anomalies`: Get recent anomaly events.
- Unit tests for all endpoints using a mock or test database.

- Run `pytest tests/unit/test_api.py`.

### Manual Verification
- Start the API: `python src/api/app.py`.
- Use `curl` or a browser to visit `http://localhost:5000/api/patients` and verify data is returned from the database.
