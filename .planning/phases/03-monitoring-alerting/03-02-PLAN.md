# Plan 03-02: Dashboard Interface

Build a visually appealing, interactive dashboard to visualize real-time patient vitals and highlight detected anomalies.

## Must Haves
- React (TypeScript) frontend application.
- Real-time (polling) visualization of vitals using Chart.js.
- Patient selector to switch between monitored individuals.
- Sidebar or notification area for recent anomalies.
- Clean, clinical-grade aesthetic using Vanilla CSS.

## Tasks

<task id="03-02-00" wave="0">
Initialize the React application in a `ui/` directory.
- Use Vite for a fast development setup.
- Install `chart.js`, `react-chartjs-2`, and `lucide-react`.
</task>

<task id="03-02-01" wave="1">
Implement core UI components:
- `PatientSelector`: Dropdown to choose from active patients.
- `PatientChart`: Line chart for vitals and anomaly scores.
- `AlertSidebar`: List of recent anomalies from the `/api/anomalies` endpoint.
</task>

<task id="03-02-02" wave="1">
Update Flask API (`src/api/app.py`) to serve the built frontend in production (optional for prototype, but good for completeness).
</task>

<task id="03-02-03" wave="2">
Integration & Polling:
- Implement data fetching with a 2-second polling interval.
- Ensure the UI updates smoothly as new data arrives.
</task>

## Verification Criteria

### Automated Tests
- Basic smoke tests for React components (e.g., rendering without crashing).

### Manual Verification
- Start the API and the UI development server.
- Verify that selecting a patient updates the chart with their specific data.
- Confirm that the chart visually distinguishes normal vs. anomalous points (or high scores).
- Verify the Alert Sidebar updates when new anomalies are detected.
