# Plan 01-02: Schema Design

Define and apply the clinical-grade relational schema optimized for time-series physiological data.

## Must Haves
- `vitals` hypertable with a 1-day chunk interval and `anomaly_score` column (ML-04).
- `anomalies` and `alerts` tables for event-based data.
- Audit trail columns (`received_at`, `processed_at`) on all core tables (`vitals`, `anomalies`, `alerts`).

- Verify that `anomalies` and `alerts` tables have the correct foreign keys and audit trail timestamps (`received_at`, `processed_at`).
