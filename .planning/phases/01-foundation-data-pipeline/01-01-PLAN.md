# Plan 01-01: Infrastructure Setup

Establish the core data infrastructure using Docker Compose to provide Apache Kafka (KRaft) and PostgreSQL with TimescaleDB.

## Must Haves
- Operational Kafka cluster in KRaft mode (no Zookeeper).
- PostgreSQL 16 instance with TimescaleDB extension enabled.
- SSL configuration for PostgreSQL to ensure secure clinical data transport.
- Python test environment with connectivity fixtures for both services.

## Tasks

<task id="01-01-00" wave="0">
Install test dependencies (`pytest-postgresql`, `pytest-docker`, `confluent-kafka`, `psycopg[binary]`) and initialize `tests/conftest.py` with shared fixtures.
</task>

<task id="01-01-01" wave="1">
Create `docker-compose.yml` defining the Kafka (KRaft) and TimescaleDB services with persistent volumes, health checks, and SSL certificate volume mounts.

</task>

<task id="01-01-03" wave="1">

</task>

<task id="01-01-04" wave="1">
Implement `tests/integration/test_kafka_connection.py` to verify the Python client can produce and consume a test message.

<task id="01-01-05" wave="1">

## Verification Criteria

### Automated Tests
- Run `docker compose up -d` to start services.
- Execute `pytest tests/integration/test_kafka_connection.py` to confirm Kafka connectivity.
- Execute `pytest tests/integration/test_db_connection.py` to confirm TimescaleDB availability and SSL enforcement.

### Manual Verification
- Inspect `docker ps` to ensure both containers are healthy.
- Verify that Kafka logs show KRaft controller is active and stable.
- Confirm SSL status via `psql` command: `\conninfo` should show SSL connection.
